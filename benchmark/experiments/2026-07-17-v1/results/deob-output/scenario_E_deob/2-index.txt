## Function Entry Format
name | Ss/Pp | cc=N | → callees ⇐ callers | closure: v1, v2 ; paramRoles ; [semTags] ; flags

## Fields
Ss/Pp       = S statements, P parameters
cc=N        = cyclomatic complexity (branch density, higher = more branches)
→ fn        = calls function fn
⇐ fn        = called by function fn
root        = entry point (no callers)
closure: v  = captures variable v from outer scope (closure capture)

## Parameter Roles (paramRoles)
this=x      = parameter used as this context
cb=x        = parameter called as callback function
out=x       = parameter properties are assigned to
arg=x       = ordinary argument
ret=x       = used in return value construction

## Semantic Tags [semTags]
[returns value]        = function has a return value
[returns via N paths]  = N distinct return paths
[returns conditional]  = conditional return
[returns str/prop]     = returns string / property
[void, NS]             = no return value, N statements
[callback-driven]      = parameters are called as callbacks
[can throw]            = may throw exceptions
[side-effects]         = has side effects (I/O, DOM, mutation)
[config]               = configuration-related
[calls → fn]           = directly invokes fn

## Flags
FLAT         = control-flow flattened (while+switch dispatcher)
DATA         = heavy hex data (large constant arrays)
[Label]      = security alert (see 1-structure.md)

## Separators
|  = field separator (name / size / cc / edges / extras)
,  = list separator within a field (multiple callees, multiple roles)
;  = extras separator (between closure, paramRoles, semTags, and flags)
